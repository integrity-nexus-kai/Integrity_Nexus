# Purpose

The eight-repository research architecture requires explicit interface boundaries.

This document defines, for each repository:

```text
1. canonical imports,
2. canonical exports,
3. allowed interface objects,
4. forbidden cross-repository claims,
5. interface maturity status.
```

The goal is to prevent accidental cross-repository claim escalation or source-of-truth confusion.

# Global Interface Rules

## Rule I1 — Imports Do Not Promote Claims

```text
When a repository imports an object from another repository, the imported object retains its source maturity and claim boundaries.
```

Forbidden:

```text
Imported candidate becomes derived result.
```

---

## Rule I2 — Exports Must Be Typed

Every exported object must be typed as one of:

```text
canonical definition
candidate
working assumption
audit result
registry status
roadmap item
governance rule
public summary
```

Untyped export is not canonical.

---

## Rule I3 — Source-of-Truth Remains Local Unless Registered

```text
A repository remains authoritative only for its declared object class.
```

Cross-repository use does not automatically transfer authority.

---

## Rule I4 — Public Presentation Never Becomes Source of Truth

```text
The public website may present bounded summaries but may not define scientific source-of-truth objects.
```

---

## Rule I5 — Controller Interfaces Are Operational, Not Physical

```text
Structural_State_Controller may export operational transition/enforcement concepts.
```

Forbidden:

```text
controller transition rule = physical Hamiltonian
```
