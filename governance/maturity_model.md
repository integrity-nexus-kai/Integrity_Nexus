# Maturity Model

This document defines the Integrity Nexus maturity model for connected research repositories.

The model is used to classify repository readiness, governance quality, formal depth, and publication maturity.

---

# M0 — Raw Notes

Status:

```text
Unstructured research material
```

Characteristics:

- no stable README,
- no governance layer,
- no registry,
- no claim boundary,
- no citation metadata.

---

# M1 — Basic Repository

Status:

```text
Basic project structure
```

Characteristics:

- README exists,
- basic files exist,
- research direction is visible,
- but governance and registry structures are incomplete.

---

# M2 — Canonical Research Skeleton

Status:

```text
Professional research structure established
```

Characteristics:

- README,
- executive summary,
- architecture map,
- governance layer,
- citation metadata,
- limitation statement,
- registry structure,
- and roadmap exist.

---

# M2.5 — Formalization Initiated

Status:

```text
Initial formal objects introduced
```

Characteristics:

- core definitions exist,
- initial formal models exist,
- registry dependencies are mapped,
- but no complete formal system is claimed.

---

# M2.8 — M3 Layer Drafted

Status:

```text
Formal specification layer drafted
```

Characteristics:

- verifier or operator models exist,
- enforcement or proof-obligation structures exist,
- expanded dependency graph exists,
- additional invariants or formal constraints exist,
- but formal soundness remains open.

---

# M3 — Formal Specification

Status:

```text
Formal framework specified
```

Characteristics:

- core calculus or formal specification exists,
- domain objects are defined,
- admissibility or validity criteria are explicit,
- proof obligations are registered,
- and unresolved formal gaps are clearly marked.

---

# M4 — Validation Layer

Status:

```text
Proof, simulation, benchmark, or implementation validation active
```

Characteristics:

- simulations,
- benchmarks,
- proof sketches,
- prototype implementations,
- or formal verification artifacts exist.

---

# M5 — Publication Ready

Status:

```text
Publication-oriented research artifact
```

Characteristics:

- paper draft exists,
- bibliography is stable,
- claims are classified,
- limitations are explicit,
- results are reproducible or proof-scoped,
- and citation metadata is complete.

---

# Governance Rule

Maturity claims must remain conservative.

A repository may not be promoted beyond the maturity level supported by its visible artifacts.
