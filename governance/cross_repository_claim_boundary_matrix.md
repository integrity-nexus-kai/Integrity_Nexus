## 1. Purpose

This file defines how claims may move between the canonical TIG research repositories without changing their status.

It is a governance artifact. It does not create theory, solve open questions, define new physics, or upgrade any result.

The protected rule is simple:

> A claim may move between repositories only if its scope, maturity, and status remain explicit.

## 3. Universal Boundary Rules

| Claim Form | Must Not Be Promoted To | Required Label |
|---|---|---|
| Candidate | Final result | Candidate only; not final |
| Compatible | Derived | Compatible only; derivation open |
| Declared | Established | Declared only; support still required |
| Operationally addressed | Scientifically solved | Operational closure only; scientific status separate |
| Effective description | Fundamental structure | Effective/scoped realization only |
| Mathematical object | Physical object | Mathematical unless separately bridged |
| Registry entry | Truth claim | Registration is governance metadata only |
| Canonical artifact | Completed theory | Canonical document status does not imply scientific completion |
| Shared term | Identical definition | Shared orientation only unless type identity is shown |

**Allowed transfer:** FRQ/OQ entries, governance rules, dependency maps, claim-boundary requirements.

**Forbidden escalation:** A Nexus organizing question must not become a TIG-E substrate, axiom, field equation, or completed derivation.

**Required boundary:** TIG-E projections remain questions, constraints, or governance objects unless locally supported.

## 6. Review Checklist

Before any cross-repository claim is accepted, check:

- Is the source repository role explicit?
- Is the target repository role explicit?
- Is the claim status preserved?
- Is the claim being upgraded silently?
- Is a mathematical object being treated as physical?
- Is operational status being treated as scientific resolution?
- Is an effective realization being treated as fundamental?
- Is a registry entry being treated as truth?
- Is shared terminology being used with domain-specific boundaries?
- Does the transfer require a bridge document?

---

## 7. Mandatory Boundary Phrases

Use where applicable:

- Candidate only; not final.
- Compatible only; not derived.
- Declared only; not established.
- Operationally addressed; scientifically open.
- Effective/scoped realization; not fundamental structure.
- Mathematical object; not physical object without bridge.
- Registered; not canonized as truth.
- Canonical artifact; not completed theory.
- Shared term; not identical definition across domains.
