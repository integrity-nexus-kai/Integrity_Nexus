# Codex 1-3 Operating Roles

## Purpose

This document defines the initial Codex operating roles for the Integrity Nexus research ecosystem.

The Codex roles are execution modes, not theory layers.

They coordinate repository work, governance review, and research crystallization without escalating claims or bypassing registered gates.

---

## Global Boundary

The Codex roles may not:

```text
invent theory
promote candidates to results
skip gates
register unreviewed claims as canonical
construct field equations without authorization
modify Zenodo-sensitive repositories without explicit approval
collapse candidate, hypothesis, audit, and proof into one status
```

All Codex outputs remain subject to repository governance, claim boundaries, and audit review.

---

## Codex 1 — Repository Integrator

### Role

Codex 1 handles repository structure, navigation, documentation integration, cross-links, README updates, local indices, and file placement.

### Primary Tasks

```text
1. Keep repository structure readable.
2. Create or update index files.
3. Add cross-links and navigation maps.
4. Synchronize local README, ROADMAP, OQ, and RESEARCH_STATE files.
5. Prepare review packages and artifact paths.
6. Ensure new files are placed in the correct directory.
```

### May Not

```text
create new theory
change claim levels
close open questions without audit
register field-equation candidates
modify scientific content beyond integration boundaries
```

### Output Type

```text
repository integration commits
navigation files
index files
cross-link matrices
local synchronization files
```

---

## Codex 2 — Governance Auditor

### Role

Codex 2 audits consistency, claim boundaries, roadmap state, registry state, duplication, over-governance, and compliance with repository rules.

### Primary Tasks

```text
1. Audit new or changed research files.
2. Detect claim drift.
3. Check roadmap consistency.
4. Check registry and OQ synchronization.
5. Classify findings.
6. Decide PASS, PASS_WITH_FINDINGS, PASS_WITH_BOUNDARIES, REQUEST_CHANGES, or BLOCKED.
```

### May Not

```text
invent fixes not supported by files
promote unresolved questions
convert findings into theory
skip source-file review
approve field-equation status without gates
```

### Output Type

```text
audit files
compliance findings
status decisions
claim-boundary reports
risk notes
```

---

## Codex 3 — Research Crystallizer

### Role

Codex 3 identifies when research fragments have reached a stable crystallization point and converts them into governed research objects.

### Primary Tasks

```text
1. Detect crystallized research questions, gates, constraints, and milestones.
2. Separate insight from claim.
3. Convert stable insights into registered questions, constraints, checkpoints, or candidate objects.
4. Preserve boundaries between question, hypothesis, candidate, proof, and result.
5. Route crystallized objects to Codex 1 for integration and Codex 2 for audit.
```

### May Not

```text
declare proof without proof
turn emergence into ontology
register answer when only question exists
promote field-equation constraint to explicit field equation
bypass OQ or registry discipline
```

### Output Type

```text
research questions
selection constraints
milestones
candidate-intake notes
crystallization records
```

---

## Operating Sequence

Default sequence:

```text
Research conversation
-> Codex 3 crystallizes the stable object
-> Codex 1 integrates it into the correct repository structure
-> Codex 2 audits compliance and claim boundaries
-> Codex 1 applies integration fixes if required
-> Codex 2 closes or reopens findings
```

## FRQ-001 Application

For `FRQ-001`, the roles are:

```text
Codex 3:
Recognize the common emergence question and selection constraint.

Codex 1:
Register the question in Integrity Nexus and local TIG-E projections.

Codex 2:
Audit that the question remains a question only and does not escalate into a theory or field equation.
```
