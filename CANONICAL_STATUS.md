# Canonical Status

## Purpose

This document defines the canonical status of the Integrity Nexus repository.

It clarifies the difference between canonical Integrity Nexus material and independent further development inspired by Integrity Nexus.

---

## Canonical Repository

The canonical Integrity Nexus repository is maintained at:

```text
https://github.com/integrity-nexus-kai/Integrity_Nexus
```

Author and repository architect:

```text
Kai Stefan Dietrich
Independent Researcher
```

---

## Canonical Rule

Only author-approved changes in the canonical repository are canonical Integrity Nexus material.

External forks, mirrors, summaries, adaptations, or derivatives are not canonical unless explicitly authorized by Kai Stefan Dietrich.

---

## Modification Boundary

The canonical material may not be modified and redistributed as if it were still the original Integrity Nexus material.

This protects:

- authorship,
- attribution,
- repository integrity,
- governance meaning,
- claim boundaries,
- and research architecture coherence.

---

## Independent Further Development

Independent further development is allowed when it is clearly marked as independent and non-canonical.

Allowed examples:

- citing Integrity Nexus,
- discussing or auditing Integrity Nexus,
- building an independent system inspired by Integrity Nexus,
- creating separate tools or research programs that reference Integrity Nexus,
- developing new ideas that acknowledge Integrity Nexus as source or inspiration.

Required boundary:

```text
Independent work must not present itself as the canonical Integrity Nexus repository.
```

---

## Naming Rule

The name Integrity Nexus should refer to the canonical repository unless explicit authorization is given.

Derivative projects should use a distinct name and state their relationship clearly.
