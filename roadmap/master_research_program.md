### P8 — AXIOMA Architecture Website and Navigation Layer

Status: planned  
Priority: critical  
Repository: Integrity Nexus / future website layer

Objective:

```text
Define the AXIOMA website as the public navigation interface for the full research ecosystem.
```

Strategic Rationale:

```text
The website should not function as a simple marketing page.

It should function as the graphical navigation surface of the AXIOMA architecture.

Integrity Nexus remains the canonical meta-repository and architectural source of truth.

The website becomes the public-facing explorer for navigating SSC, TIG-E, SIR, Research OS, repositories, publications, and investor-facing material.
```

Architecture Role:

```text
Website = graphical user interface / navigation layer
GitHub = canonical file system / repository backend
Integrity Nexus = architectural control layer
SSC = integrity and AI-alignment layer
TIG-E = scientific execution and physics research layer
SIR = recursive governance layer
Research OS = execution and audit infrastructure
```

Primary Navigation Model:

```text
HOME
|
|-- Why AXIOMA?
|-- Architecture
|-- Research
|-- Technology
|-- Investors
|-- Repositories
|-- Publications
|-- Community
```

Architecture Page Role:

```text
The Architecture page should become the dominant entry point for understanding AXIOMA.

It should present AXIOMA as an integrity-preserving knowledge-generation architecture, not merely as a collection of repositories.

Visitors should be able to navigate by interest:

- scientists -> TIG-E and research papers,
- AI researchers -> SSC and alignment governance,
- developers -> repositories and tooling,
- investors -> Research OS, platform strategy, and commercial roadmap,
- reviewers -> Integrity Nexus, audits, dependencies, and maturity dashboards.
```

Canonical Message:

```text
AXIOMA is an integrity-preserving architecture for scientific discovery, AI alignment, and research governance.
```

Website Claim Boundary:

```text
The website may describe architecture, roadmap, repository structure, governance, research status, and intended applications.

It may not present unresolved research claims as completed results.

It may not present TIG-E field-equation material as validated physics before external review.

It must preserve the same claim-boundary discipline as the repositories.
```

Milestones:

```text
M1 — Draft AXIOMA Architecture Whitepaper in Integrity Nexus.
M2 — Define website information architecture and page hierarchy.
M3 — Create architecture diagram linking SSC, TIG-E, SIR, Research OS, and applications.
M4 — Define audience-specific navigation routes.
M5 — Connect public website pages to canonical GitHub repository documents.
M6 — Add investor-facing platform narrative without weakening scientific claim boundaries.
M7 — Publish website as public navigation layer after core repository consolidation.
```
