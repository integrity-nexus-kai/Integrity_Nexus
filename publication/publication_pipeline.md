# Publication Pipeline

## Purpose

This document defines the publication workflow for research artifacts governed through Integrity Nexus.

---

## Pipeline Stages

```text
P0 — Idea / notes
P1 — Repository scaffold
P2 — Technical outline
P3 — Draft manuscript
P4 — Internal audit
P5 — External-ready preprint
P6 — Submission / release
P7 — Post-publication maintenance
```

---

## Workflow

1. Register the publication candidate.
2. Identify source repository and claim scope.
3. Confirm bibliography and citation metadata.
4. Confirm limitations and non-claims.
5. Run pre-publication audit.
6. Prepare release or submission package.
7. Update repository and Nexus status.

---

## Mandatory Checks

Every publication candidate must pass:

- claim-boundary review,
- limitation review,
- citation review,
- dependency review,
- and maturity-status review.
