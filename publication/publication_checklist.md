# Publication Checklist

## Purpose

This checklist must be completed before a publication candidate is externally released.

---

## Required Checks

### Repository Check

- [ ] Source repository identified.
- [ ] Repository status is current.
- [ ] Maturity level is current.
- [ ] README and executive summary are current.

### Claim Check

- [ ] Claims are classified.
- [ ] Limitations are explicit.
- [ ] Non-claims are visible.
- [ ] Cross-repository claims are bounded.

### Evidence Check

- [ ] Evidence is separated from interpretation.
- [ ] Bibliography is present.
- [ ] External references are current.
- [ ] No unsupported result is implied.

### Governance Check

- [ ] Development status is updated.
- [ ] Audit findings are reviewed.
- [ ] Dependencies are mapped.
- [ ] Open problems are registered.

### Release Check

- [ ] Version or release label assigned.
- [ ] Citation metadata present.
- [ ] License present.
- [ ] Nexus repository status updated.
