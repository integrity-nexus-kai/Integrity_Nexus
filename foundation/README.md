# Foundation Layer

The Foundation Layer defines the universal discovery architecture used by TIG-E before any domain-specific application is attached.

It is the kernel of the TIG-E discovery engine inside the Integrity Nexus meta-repository.

It does not describe physics, chemistry, medicine, AI alignment, psychology, materials science, or any other specific domain. It defines the meta-structure by which complex search spaces are decomposed, governed, compressed, reviewed, and converted into auditable research states.

## Repository Role

```text
Integrity Nexus = meta-repository
Foundation Layer = universal discovery kernel
Domain Packs = concrete applications
Operations = execution workflows
Registry = structured research state
Governance = canonical rules and claim boundaries
```

The Foundation Layer defines the principles. It does not execute them by itself.

Execution belongs to Operations. State belongs to Registry. Claim-boundary protection belongs to Governance. Domain-specific application belongs to Domain Packs.

## Canonical Scope

Foundation documents define the kernel of the TIG-E discovery architecture:

1. `01_foundation_index.md` — overall map, layer separation, and document order.
2. `02_what_is_a_generator.md` — generator/projection distinction, explanation depth, and provisional stop condition.
3. `03_search_space_compression.md` — how TIG-E reduces complex search spaces and clarifies structure.
4. `04_emergence_protocol.md` — how controlled emergence is prepared, not assumed.
5. `05_fundamental_candidate_admission.md` — admission rules for candidate structures.
6. `06_discovery_engine_architecture.md` — universal discovery engine architecture.
7. `07_operations_protocol.md` — operational workflow and role separation.

## Core Principle

```text
Foundation describes the universal principles of discovery.
Domain packs describe their application to concrete problem spaces.
```

## Separation Rule

```text
No domain-specific projection may be promoted into the Foundation Layer as if it were universal.
```

The physics program, including the Integrity Field and any fundamental field-equation work, is a domain application of this Foundation Layer. It does not define the Foundation Layer itself.

## Current Status

The Foundation Layer is initialized as a v0.1 architecture layer.

The documents are canonical only as current structural guidance, not as final theory, final ontology, or final automation specification.
