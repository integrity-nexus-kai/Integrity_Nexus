# Foundation Index

## Purpose

This document defines the Foundation Layer of the TIG-E discovery architecture inside the Integrity Nexus meta-repository.

The Foundation Layer is not a physics theory, not a domain pack, and not a software implementation. It is the universal kernel for structured discovery, research governance, search-space compression, controlled emergence, and auditable candidate handling.

## Layer Separation

```text
Foundation Layer
    universal discovery principles
        ↓
Domain Packs
    physics, pharma, AI alignment, psychology, materials, etc.
        ↓
Operations / Registry / Governance
    workflow, state, review, automation, canonical protection
        ↓
Concrete Research Artifacts
    candidates, open questions, proofs, simulations, publications
```

The first reference domain is physics. The physics domain may introduce terms such as Integrity Field, field equation, spacetime, metrics, tensors, or operators.

Those terms do not belong to the universal Foundation Layer unless explicitly abstracted into a domain-neutral governance role.

## Foundation Documents

1. **What is a Generator?**
   - Defines generator vs. projection.
   - Defines the explanation-depth criterion.
   - Defines the provisional stop condition for sufficient explanation.

2. **Search Space Compression**
   - Defines how TIG-E reduces complex search spaces without prematurely selecting answers.
   - Defines the requirement that each iteration either compresses the search space or clarifies its structure.

3. **Emergence Protocol**
   - Defines how controlled emergence is prepared by structure, not claimed by assertion.
   - Treats emergent outputs as reviewable candidates, not final results.

4. **Fundamental Candidate Admission Criteria**
   - Defines how candidate structures enter, leave, or remain outside a fundamental candidate pool.
   - Requires generator/projection classification.

5. **Discovery Engine Architecture**
   - Defines TIG-E as a universal discovery engine.
   - Separates the engine from its domain applications.

6. **Operations Protocol**
   - Defines the operational workflow: input, structuring, filters, registry, review, GitHub Actions, and repository integration.

## Non-Goals

The Foundation Layer does not:

- prove a physical theory,
- define the Integrity Field mathematically,
- derive a field equation,
- privilege one domain over another,
- replace human research judgment,
- automate final truth claims,
- or convert a domain-specific concept into a universal primitive without review.

## Primary Rule

```text
No domain-specific projection may be smuggled into the universal Foundation Layer as if it were foundational.
```

This rule protects TIG-E from confusing a reference implementation with the discovery architecture itself.

## Current Status

The Foundation Layer is initialized as a v0.1 architecture layer.

All documents are provisional, reviewable, and subject to refinement through the same TIG-E governance logic they define.
